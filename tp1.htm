TP1 – Système de messagerie SmartMail...

INF2160 – Automne 2018

 

Description

Le but de ce travail est de créer une application qui simule un système de gestion de messagerie électronique appelé SmartMail. SmartMail permet à un utilisateur inscrit de disposer d’un service de messagerie électronique offrant trois boîtes (Envoi, Réception et Spams). Les messages suspects sont automatiquement envoyés dans la boîte Spams. Ils sont filtrés selon un ensemble de principes ou d’heuristiques spécifiés dans le fichier TP1.hs. L’utilisateur peut aussi entretenir un ensemble de contacts et éventuellement de préférences. L’état d’un contact suspect passera de « blanc » à « noir » indiquant ainsi son inscription dans la liste noire et provoquant une redirection automatique des messages reçus de ce contact dans la boîte Spams. Un service d’administration de SmartMail est offert pour la gestion des messages et la production de quelques statistiques d’intérêt.

 

Ressources

Ce travail comporte trois fichiers :

ü  le fichier TP1.hs que vous devez compléter (il comporte une descriptions détaillées des fonctions à réaliser),

ü  le fichier Tp1DataTypes.hs, un module qui comporte les définitions de types et synonymes de types nécessaires pour l’application,

ü  et le fichier  testTP1.hs (qui vous sera fourni ultérieurement pour les tests de validation). Il comportera des tests (non exhaustifs) qui doivent retourner True.

 

Ce que vous devez faire 

ü  Compléter le fichier TP1.hs en lisant attentivement les descriptifs donnés en commentaire et produire une solution satisfaisante.

ü  Tester progressivement vos fonctions à travers l’application de la fonction main définie dans le fichier TP1.hs. Cette fonction sans paramètre est un point d’entrée de l’application offrant un menu qui vous permettra de gérer le système de messagerie aussi bien en tant qu’utilisateur qu’administrateur. Vous pouvez télécharger l’exécutable TP1.EXE (ou TP1 pour OS X; ou TP1 pour Linux)) pour visualiser le comportement de certaines fonctions (utiliser le nombre 12345 comme mot de passe pour l’accès l’option ‘administrateur’ du menu). Les utilisateurs suivants sont déjà dans le système : nkambou.roger@smail.ca et tato.ange@smail.ca. Lors de l’envoi d’un message, ne contenant que le caractère ‘&’ au début et rien d’autre marque la fin du message.

ü  Tester vos solutions en utilisant les tests (non exhaustifs) donnés dans le fichier testTP1.hs. Vous devez utiliser ce fichier de la manière suivante (une fois que votre travail est prêt):

    • Compilez-le comme suit (dans le même répertoire que celui de votre TP1.hs): ghc –make testTP1.hs
    • Exécutez-le en tapant testTp1.

 

Les bonnes pratiques de génie logiciel suggèrent d'utiliser les fonctions disponibles dans les librairies du langage lorsque c'est possible. Vous pouvez également définir vos propres fonctions de soutien (ou de service) qui vous aideront à répondre aux questions posées.
